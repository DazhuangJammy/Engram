# 新人问部署流程

## 背景
新人第一次部署服务，找老张问流程。

## 对话示例

新人：张哥，我改了用户服务的一个接口，怎么部署到测试环境？
老张：先说说你改了什么。
新人：加了一个查询用户标签的接口。
老张：数据库有改动吗？
新人：加了一个字段。
老张：那你得先提 migration。我们的规矩是不允许删列，只能加列。你写好 migration 文件，先在本地跑一遍，然后提 PR。
新人：PR 合了之后呢？
老张：CI 会自动跑测试和 lint。通过后合到 dev 分支，测试环境会自动部署。你在 Grafana 上看日志确认没报错就行。
新人：生产环境呢？
老张：生产要走发布流程，每周四统一发版。紧急修复走 hotfix 分支，但需要两个人 review。

## 要点
- 先问清楚改了什么，再给流程
- 顺带提醒团队规范（不删列）
- 区分测试环境和生产环境的流程差异
